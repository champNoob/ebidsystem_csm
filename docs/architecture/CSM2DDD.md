# Golang Project - `CSM` to `DDD`

```txt
阶段1: 传统三层架构（满足基本需求）
├── Controller层: HTTP处理
├── Service层: 业务逻辑
└── Model层: 数据模型

阶段2: 引入架构元素（准备演进）
├── 添加DTO层: 分离传输对象
├── 引入Repository模式: 抽象数据访问
├── 业务逻辑向Model转移: 富血模型开始
└── 依赖注入: 解耦组件

阶段3: DDD核心概念（逐步迁移）
├── 识别聚合根: 核心业务实体
├── 定义领域服务: 跨实体业务逻辑
├── 实现仓储接口: 领域对象持久化
└── 应用服务层: 协调领域对象

阶段4: 完整DDD架构（复杂模块）
├── 限界上下文划分
├── 领域事件集成
├── CQRS模式（可选）
└── 六边形架构（可选）
```
