# 信息泄露（Information Disclosure）与业务错误封装

## 案例

当前端调用 `CreateUser` 接口注册用户时，后端返回了响应文本：“`Error 1062 (23000): Duplicate entry 'Cindy721' for key 'users.uni_users_username'`”，但是前端界面竟然也显示了上述响应文本。

类似的，如果前端请求中的 `password` 位数不符合后端业务逻辑，后端也会返回相应的文本并在前端页面显示——这显然不合理。

## 隐患

后端暴露了：

- 数据库类型（MySQL）
- 具体错误码（1062）
- 表名、索引名
- 甚至可能暴露字段内容

## 解决方法

> 后端封装业务错误，前端只显示“用户可理解的错误”
